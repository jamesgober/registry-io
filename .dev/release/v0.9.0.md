# Release v0.9.0 — Documentation Pass (Pre-RC Gate)
**Date:** 2026-05-19
**Compare:** `v0.8.0...v0.9.0`

## Summary

This release is the final documentation pass before the
`1.0.0-rc.1` cut. Three new long-form docs land — the v1.0
stability contract, the internal architecture walk-through, and
the per-platform behavior notes — and the public rustdoc surface
is now audit-complete: every non-trivial public item carries at
least one runnable `# Examples` block, verified by 46 passing
doctests (up from 26).

Phase **0.9.0**'s documentation portion is complete. The
remaining phase-0.9.0 items — cutting `1.0.0-rc.1`, the 1-week
soak, the 1-CPU-hour fuzz soak deferred from 0.7.0 — are

is published.

## Added

### Long-form documentation (3 new files)

- **`docs/STABILITY-1.0.md`** — the v1.0 stability contract.
  Enumerates every public item that will be frozen in v1.x with
  its behavioral guarantees: lock-free + zero-allocation
  `SyncRegistry::notify`, priority ordering with stable ties,
  panic isolation as default, `HandlerId` uniqueness, `Send + Sync`
  on every public type. Pins MSRV `1.85`, edition `2024`, license
  `Apache-2.0 OR MIT`. Specifies what can still evolve in v1.x
  (additions, internal layouts, dependency bumps inside semver),
  the deprecation policy (announce in minor, remove in next major),
  and the yank policy. Lists the CI gates and test files that
  enforce each contract clause.
- **`docs/ARCHITECTURE.md`** — internal structure walk-through.
  Opens with an ASCII big-picture diagram of `SyncRegistry<E>`'s
  storage model (`ArcSwap<Vec<HandlerEntry<E>>>` +
  `HandlerIdGenerator` + `ArcSwapOption<PanicCallbackHolder>`).
  Full file-tree map of `src/`, `tests/`, `benches/`, `examples/`,
  `fuzz/`. Per-section walk: storage model, the hot path
  (`notify` decomposed with measured per-stage costs), the slow
  path (rcu pattern + priority-sorted insertion via
  `Vec::partition_point`), the async path (`BoxFuture`,
  `CatchUnwind`, `JoinAll`, the two dispatch modes), RAII guards
  (`Weak`-not-`Arc` rationale), and a "Cross-cutting design
  decisions" section explaining why each major choice was made
  (`E: Send + Sync + 'static` at the type level, monotonic ids,
  panic isolation by default, no embedded async runtime). Closes
  with an "Adding a new feature" 7-step contributor checklist.
- **`docs/PLATFORM-NOTES.md`** — per-platform behavior nuances.
  Support matrix: Linux/macOS/Windows on stable + MSRV gated;
  WASM and `no_std` explicitly listed unsupported with the reason
  (`ArcSwap`/`std::panic`/`std::sync::Arc` are required and not
  validated outside std-on-x86/aarch64). Per-platform notes
  including the Windows SEH unwind characteristics that make
  `catch_unwind` ever-so-slightly more expensive than on Linux
  (still inside the contract). Build prerequisites per platform.
  CI matrix walk-through citing the actual workflow file. Known
  quirks: `dhat::Profiler` is process-global (and why
  `tests/zero_alloc.rs` collapses two scenarios into one test);
  stable-Rust feature gates (no `#![feature(...)]` anywhere);
  allocator-interaction notes for users running `jemalloc` /
  `mimalloc`. Closes with the bug-reporting checklist for
  platform-specific issues.

### Rustdoc completeness (20 new doctest blocks)

Every non-trivial public item now has at least one runnable
`# Examples` block. Items that received new examples in this
release:

- `pub const VERSION` — diagnostic-use example.
- `AsyncRegistry::register_guard_with_priority` — high + low
  priority guard registration example.
- `AsyncRegistry::clear` — register five, clear, assert empty.
- `AsyncRegistry::handler_count` — 0 → 1 transition example.
- `AsyncRegistry::is_empty` — same shape.
- `AsyncRegistry::contains` — register, contains, unregister,
  not-contains.
- `AsyncRegistry::on_panic` — full async-fn example with
  `#[tokio::main]` runtime, atomic panic-counter callback.
- `AsyncRegistry::clear_panic_callback` — install-then-clear
  example.

All 46 doctests pass under `cargo test --doc --all-features`.
