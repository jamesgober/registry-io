# Release v0.5.0 — Asynchronous Registry
**Date:** 2026-05-19
**Compare:** `v0.4.0...v0.5.0`

## Summary

This release adds the **asynchronous** side of the registry: `AsyncRegistry`,
behind the `async` feature flag, with the same lock-free storage model as
`SyncRegistry` and identical panic-isolation guarantees adapted to the
async path. Two dispatch modes are exposed — concurrent (default `notify`)
and sequential (`notify_sequential`) — and the supporting future
combinators are written in-tree to avoid pulling in `futures-util`.

Roadmap phase **0.5.0** is complete with this release. Phase 0.6.0
(performance verification) is next.

## Added

### Core types

- **`AsyncRegistry<E>`** (`src/async_registry/mod.rs`) — async-handler
  counterpart to `SyncRegistry`. Generic over `E: Send + Sync + 'static`.
  Stores handlers as `Arc<dyn Fn(&E) -> Pin<Box<dyn Future<Output = ()> + Send + 'static>> + Send + Sync + 'static>`
  behind an `arc_swap::ArcSwap<Vec<AsyncHandlerEntry<E>>>`. Public API:
  - `new()`, `with_capacity(usize)`, `Default`, `Debug`.
  - `register(handler) -> HandlerId` (default priority `0`).
  - `register_with_priority(priority, handler) -> HandlerId`.
  - `register_guard(handler) -> AsyncHandlerGuard<E>` (requires `Arc<Self>`).
  - `register_guard_with_priority(priority, handler) -> AsyncHandlerGuard<E>`.
  - `unregister(id) -> bool`, `clear()`, `contains(id) -> bool`,
    `handler_count() -> usize`, `is_empty() -> bool`.
  - `on_panic(callback)`, `clear_panic_callback()` (shared `PanicInfo`
    machinery with `SyncRegistry`).
  - `notify(&event) -> impl Future<Output = ()>` — concurrent dispatch via
    crate-local `JoinAll`.
  - `notify_sequential(&event) -> impl Future<Output = ()>` — sequential
    dispatch in priority order.
- **`AsyncHandlerGuard<E>`** (`src/async_registry/guard.rs`) — RAII
  guard for async registrations. `#[must_use]`, drop-to-unregister,
  `forget()` to detach. Holds `Weak<AsyncRegistry<E>>` so
  registry-before-guard drop is safe.

### Future combinators (in-tree)

- **`CatchUnwind<F>`** (`src/future_ext.rs`) — heap-pinned future adapter
  that wraps a child future's `poll` in `std::panic::catch_unwind`. Yields
  `Result<F::Output, Box<dyn Any + Send>>`. On panic the inner future is
  consumed, satisfying the no-double-poll invariant.
- **`JoinAll<F>`** (`src/future_ext.rs`) — minimal homogeneous-type future
  combinator that polls every still-`Pending` slot per wake and resolves
  to a `Vec<F::Output>` in input order. Written in-tree to keep the
  optional-dependency surface limited to `arc-swap`.

### Module wiring

- **`pub mod r#async`** in `src/lib.rs` (feature-gated, `#[path = "async_registry/mod.rs"]`),
  exposing `registry_io::r#async::{AsyncRegistry, AsyncHandlerGuard}`.
- Crate-root re-export: `pub use r#async::{AsyncHandlerGuard, AsyncRegistry}`
  when the `async` feature is on.
- `panic` module gating broadened to
  `any(feature = "sync", feature = "async")` since both registries share
  `PanicInfo` and `PanicCallbackHolder`.

### Tests

- `src/future_ext.rs` — 4 unit tests for `CatchUnwind` and `JoinAll`
  (normal completion, panic capture, ordering, empty input).
- `tests/async_registry.rs` — 10 tests: empty/notify-noop, concurrent
  notify, sequential notify, unregister, clear, by-reference event
  passing, mixed register/unregister counting, send/sync.
- `tests/async_priority.rs` — 3 tests for sequential priority ordering.
- `tests/async_panic.rs` — 11 tests covering both dispatch modes:
  non-propagation, sibling survival, callback id+message capture,
  per-handler fire counts, custom payload downcast, silent absorption,
  count stability, callback clear.
- `tests/async_guards.rs` — 7 tests: drop unregisters, id round-trip,
  `forget`, drop-order safety, drop prevents firing, multiple
  independent guards, priority guard with sequential notify.

Total async surface coverage: **35** integration tests +
**4** unit tests (= 39 new tests). Whole-crate suite: **131 passing tests**.

### Benchmarks

- `benches/async_notify.rs` — concurrent and sequential scenarios at
  handler counts `0, 1, 4, 16`. Built only when `--features async` is
  enabled (`required-features` in `Cargo.toml`).

### Examples

- `examples/async_basic.rs` — minimal end-to-end async register / notify.
- `examples/async_concurrent_vs_sequential.rs` — wall-clock comparison
  between the two dispatch modes against four 50ms-sleeping handlers.

Both examples are gated by `required-features = ["async"]`.

### Documentation

- `docs/API.md` — extended with the `AsyncRegistry` and
  `AsyncHandlerGuard` sections. Every public async method documented
  with parameters, returns, and a runnable example.
- `README.md` — async quick-start added alongside the four existing sync
  examples. Highlights and status sections updated.

## Changed

- **`Cargo.toml`** — version `0.4.0` → `0.5.0`. New `[[bench]]` for
  `async_notify` and `[[example]]` entries for the two new async examples,
  all `required-features = ["async"]`. `tokio` dev-dep now also enables
  the `time` feature (used by the wall-clock example).
- **`src/lib.rs`** — `#![doc(html_root_url)]` bumped to `0.5.0`. Module
  wiring: `panic` module gated on `any(feature = "sync", feature = "async")`;
  new `future_ext` module gated on `feature = "async"`; new
  `pub mod r#async;` with `#[path = "async_registry/mod.rs"]`; new crate-root
  re-exports for `AsyncRegistry` / `AsyncHandlerGuard`.
